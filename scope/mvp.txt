## Executive Summary

**CONSSOR Platform MVP** — A three-portal consulting marketplace powered by AI pricing, dual payment gateways, and subscription-based consultant access.

### Revenue Model

| Revenue Stream | Mechanism | Rate/Amount |
|---|---|---|
| **Engagement Fees** | 12% platform commission on every client payment | 12% of gross |
| **Consultant Subscriptions** | Weekly/Monthly/Annual recurring billing | ₹999/wk · ₹2,999/mo · ₹24,999/yr |
| **Referral Bonuses** | Paid to consultants from platform's 12% pool | 5% of converted deal value (configurable) |

### Payment Flow Summary

1. **Client Onboarding** → AI generates pricing quote (Groq API) → Admin reviews/approves
2. **Client Payment** → Stripe (India) or Stripe (International) → Payment confirmed
3. **Access Gate Unlocked** → Client gains full portal access (messaging, docs, scheduling, milestones)
4. **Commission Split** → Platform takes 12%, consultant receives 88%
5. **Payout Approval** → Admin approves consultant payout → Manual bank transfer → Marked as paid

### Consultant Subscription Model

- **Subscription required** to receive client assignments, submit leads, and use the platform
- **3 tiers:** Starter (weekly), Professional (monthly), Growth (annual)
- **Free trial:** 7 days on Starter plan, no credit card required
- **Access gate:** Expired subscription = blocked from new clients, new leads, new messages

### Technology Stack

- **AI Pricing:** Groq Sonnet 4 via Groq API (JSON output for structured pricing)
- **Payments (India):** Stripe — supports INR, UPI, cards, Net Banking
- **Payments (International):** Stripe — supports USD/EUR, cards, SEPA
- **Backend:** Python (Django) — Hosted on AWS
- **Database:** Supabase (PostgreSQL)
- **Frontend:** Next.js (React) — Hosted on Vercel

**Consultant Portal (5 views)**

- **Phase 1 (Weeks 1-4):** Core auth, onboarding wizard, AI pricing integration, database schema
- **Phase 2 (Weeks 5-8):** Payment gateways (Stripe + Stripe), access gates, subscription billing

## 3. Payment & Monetisation Architecture

> This section defines all money flows on the platform: how clients are charged, how the platform takes its cut, and how consultants pay to access the platform.

---

### 3.1 AI-Powered Engagement Pricing

After a client completes the onboarding wizard, the platform uses an AI engine to analyse the submitted scope and generate a recommended engagement price before any human interaction begins.

#### How It Works

1. Client submits the 6-step onboarding wizard (domain, scope, stage, budget, engagement type)
2. The platform passes all structured onboarding data to the AI pricing engine
3. AI analyses:
   - Scope complexity (number of service areas, breadth of problem)
   - Industry vertical (some verticals command higher rates — e.g., Financial Services vs. Retail)
   - Project stage (Ideation is lighter; M&A / Transaction Support is heavier)
   - Engagement type (One-time vs. Retainer)
   - Client-stated budget (used as a soft anchor, not a hard cap)
   - Historical data from similar past engagements (feeds back into model over time)
4. AI returns a **recommended price range** and a **suggested fixed quote**
5. Admin reviews and can override the AI quote before it is shown to the client
6. Client sees the final quote and proceeds to payment

#### AI Prompt Inputs (Structured)

```
{
  "domain_tags": ["Financial Services", "Digital Economy & ICT"],
  "engagement_scope": ["Strategic Consulting", "Digital Transformation"],
  "project_stage": "Growth / Scaling",
  "engagement_type": "Retainer",
  "budget_range": "$50,000 – $200,000",
  "description": "Redesign core banking digital product and define a 3-year tech roadmap.",
  "country": "India"
}
```

#### AI Output Structure

```
{
  "recommended_price_usd": 48000,
  "price_range_low": 38000,
  "price_range_high": 62000,
  "pricing_rationale": "Multi-scope engagement combining strategic and digital workstreams in a high-complexity vertical. Growth stage with retainer model suggests sustained delivery over 3-6 months.",
  "estimated_duration_weeks": 16,
  "confidence_score": 0.82
}
```

#### Admin Override
- Admin sees the AI quote + rationale in the Admin Panel before it reaches the client
- Can adjust the final quote up or down
- Must add a note if overriding (for audit trail)
- Once approved by Admin: quote is locked and dispatched to the client dashboard

### 3.2 Client Payment Flow & Access Gate

Payment is a **hard gate**: clients cannot access messaging, project tracker, document signing, or consultant scheduling until full payment is confirmed.

#### Payment Flow

```
Onboarding Complete
       ↓
AI Pricing Engine generates quote
       ↓
Admin reviews & approves quote
       ↓
Client receives quote on dashboard
       ↓
Client clicks "Proceed to Payment"
       ↓
Payment Gateway (Stripe / Stripe)
       ↓
Payment Confirmed → Engagement Activated
       ↓
Full platform access unlocked:
  ✅ Messaging thread
  ✅ Project tracker & milestones
  ✅ Consultant profile & scheduling
  ✅ NDA & document signing
```

#### Quote Review Screen (Client-facing)

| Field | Description |
|---|---|
| Engagement Title | Auto-generated from scope (e.g., "Digital Transformation — FinServ Growth") |
| Scope Summary | Bullet list of selected services |
| Estimated Duration | X weeks |
| Pricing Model | Fixed / Retainer |
| Total Amount | $XX,XXX |
| Platform Assurance | "Secured by CONSSOR. Dedicated consultant, milestone tracking, document signing." |

CTA: **"Pay & Unlock Engagement"** (primary) · "Request Revision" (secondary)

#### Request Revision Flow
- Client flags disagreement with the quote via free-text field
- Submission notifies Admin → Admin adjusts quote → Client re-reviews

#### Payment Gateways
- **India:** Stripe (INR, UPI, Net Banking, cards)
- **Turkey / International:** Stripe (USD / EUR, cards, SEPA)
- Currency auto-detected from client country at checkout

#### Payment Statuses

| Status | Description |
|---|---|
| `awaiting_quote` | AI quote being generated or pending admin approval |
| `quote_sent` | Client has received and can view the quote |
| `payment_pending` | Client initiated payment, gateway processing |
| `paid` | Payment confirmed, engagement fully active |
| `revision_requested` | Client asked for quote adjustment |
| `failed` | Payment gateway returned failure |
| `refunded` | Admin issued a refund |

#### Access Gate Logic

```
IF engagement.payment_status != "paid":
  BLOCK: /client/messages, /client/projects,
         /client/consultant/:id, /client/documents
  SHOW:  "Complete payment to unlock your engagement"
  SHOW:  Quote card + Pay CTA

IF engagement.payment_status == "paid":
  ALLOW: Full client portal access
```

### 3.3 Platform Commission — 12% Fee

CONSSOR charges a **12% platform fee** on every client payment, deducted before the consultant's payout is calculated.

#### Fee Calculation Example

```
Client Pays:           $48,000  (100%)
Platform Fee (12%):     $5,760  → credited to CONSSOR
Consultant Receives:   $42,240  → net payout (88%)
```

#### Fee Rules
- Applied on the **total engagement value** at time of payment confirmation
- For milestone-based retainers, fee is applied per milestone payment release
- Referral bonuses are paid from CONSSOR's 12% pool (not deducted from the consultant's 88%)
- Admin can override the fee % per engagement if needed (default: 12%)

#### Visibility
- **Admin:** Sees gross paid, platform fee, and consultant net payout
- **Consultant:** Sees only their net payout (88%)
- **Client:** Sees total amount paid; the internal fee split is not disclosed

### 3.4 Consultant Subscription Plans

Consultants pay a recurring subscription to access CONSSOR — enabling them to receive client assignments, manage leads, use earnings tools, and maintain a verified profile.

> The subscription is independent of earnings. Consultants always receive 88% of client payments regardless of plan tier. The plan governs platform access limits and feature availability.

#### Plan Comparison

| Feature | Starter | Professional | Growth |
|---|---|---|---|
| **Billing Cycle** | Weekly | Monthly | Annual |
| **Price (INR)** | ₹999 / week | ₹2,999 / month | ₹24,999 / year |
| **Price (USD approx.)** | ~$12 / week | ~$36 / month | ~$300 / year |
| **Savings vs. Starter** | — | ~25% | ~52% |
| Max Active Clients | 2 | 5 | Unlimited |
| Lead Submissions / Month | 3 | 10 | Unlimited |
| Referral Bonus Eligible | ❌ | ✅ | ✅ |
| Profile Visibility | Standard | Enhanced | Featured (priority) |
| KYC Fast-Track | ❌ | ❌ | ✅ |
| Dedicated Account Manager | ❌ | ❌ | ✅ |
| Analytics Dashboard | Basic | Standard | Advanced |
| Priority Assignment Matching | ❌ | ✅ | ✅ |
| Support | Email only | Email + Chat | Email + Chat + Phone |

#### Billing Details
- Billed via Stripe (India) or Stripe (international)
- Auto-renewal on by default; consultant can cancel anytime
- Access continues until end of the current billing period on cancellation
- **Downgrade:** Effective at next billing date
- **Upgrade:** Prorated charge applied immediately
- **Free Trial:** 7-day Starter access on signup, no credit card required (max 1 client, 2 leads)

#### Subscription States

| Status | Description |
|---|---|
| `trial` | 7-day free trial active |
| `active` | Valid subscription, full plan access |
| `past_due` | Payment failed; 3-day grace period before access lock |
| `cancelled` | Cancelled; access continues until period end |
| `expired` | Period ended, no active subscription |

#### Consultant Access Gate

```
IF consultant.subscription_status IN ["expired", "past_due" after grace]:
  BLOCK: Receiving new client assignments
         Submitting new leads
         Messaging new clients
  SHOW:  "Your subscription has expired. Renew to continue."
  SHOW:  Plan selection + payment CTA
```

#### Admin Controls
- Admin can manually grant complimentary access or override subscription status
- All consultant subscription statuses visible in the Consultant Directory
- Subscription revenue tracked as a separate line item in the Admin revenue view

# Webhook handler (Django View)
from django.views.decorators.csrf import csrf_exempt
from django.http import HttpResponse

@csrf_exempt
def stripe_webhook(request):
    payload = request.body
    sig_header = request.META.get('HTTP_STRIPE_SIGNATURE')
    endpoint_secret = settings.STRIPE_WEBHOOK_SECRET

    try:
        event = stripe.Webhook.construct_event(
            payload, sig_header, endpoint_secret
        )
    except Exception as e:
        return HttpResponse(status=400)

    if event['type'] == 'payment_intent.succeeded':
        intent = event['data']['object']
        engagement_id = intent['metadata']['engagement_id']
        
        # Update database via Django ORM
        payment = ClientPayment.objects.get(gateway_payment_id=intent['id'])
        payment.status = 'paid'
        payment.paid_at = timezone.now()
        payment.save()
        
        engagement = Engagement.objects.get(id=engagement_id)
        engagement.payment_status = 'paid'
        engagement.status = 'active'
        engagement.save()
        
        # Unlock access and notify
        unlock_client_access(engagement.client_id)
        send_payment_confirmation_emails(engagement.id)

    return HttpResponse(status=200)

from django.shortcuts import redirect
from django.urls import reverse

class ClientPaymentRequiredMiddleware:
    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request):
        if request.user.is_authenticated and request.user.role == 'client':
            # Identify protected paths
            protected_paths = ['/client/messages', '/client/projects', '/client/consultant', '/client/documents']
            if any(request.path.startswith(path) for path in protected_paths):
                # Check for active paid engagement
                has_paid_engagement = Engagement.objects.filter(
                    client=request.user,
                    payment_status='paid'
                ).exists()
                
                if not has_paid_engagement:
                    return redirect(reverse('client_payment_page'))
                    
        return self.get_response(request)

#### Consultant Subscription Gate (Django Middleware)

```python
class ConsultantSubscriptionMiddleware:
    def __init__(self, get_response):
        self.get_response = get_response

    def __call__(self, request):
        if request.user.is_authenticated and request.user.role == 'consultant':
            # Skip check for subscription and settings pages
            if request.path.startswith('/consultant/subscription') or request.path.startswith('/consultant/settings'):
                return self.get_response(request)

            subscription = ConsultantSubscription.objects.filter(consultant=request.user).order_by('-created_at').first()
            
            if not subscription or subscription.status in ['expired', 'cancelled']:
                return redirect(reverse('consultant_subscription_page'))
            
            # Handle past_due grace period
            if subscription.status == 'past_due':
                grace_period_end = subscription.current_period_end + timedelta(days=3)
                if timezone.now() > grace_period_end:
                    subscription.status = 'expired'
                    subscription.save()
                    return redirect(reverse('consultant_subscription_page'))
                    
        return self.get_response(request)
```

from django.db import transaction
from decimal import Decimal

@transaction.atomic
def process_payment_revenue(payment):
    PLATFORM_FEE_PCT = Decimal('0.12')
    
    platform_fee = payment.amount * PLATFORM_FEE_PCT
    consultant_payout = payment.amount - platform_fee
    
    payment.platform_fee_pct = PLATFORM_FEE_PCT
    payment.platform_fee_amount = platform_fee
    payment.consultant_payout_amount = consultant_payout
    payment.save()
    
    # Create payout record for consultant
    engagement = payment.engagement
    
    Payout.objects.create(
        consultant=engagement.consultant,
        period_month=timezone.now().month,
        period_year=timezone.now().year,
        gross_earnings=payment.amount,
        platform_fee_pct=PLATFORM_FEE_PCT,
        platform_fee_amount=platform_fee,
        net_payout=consultant_payout,
        referral_bonus=0, # Updated separately if applicable
        total_due=consultant_payout,
        status='pending'
    )

from django.db.models import Sum, Count
from django.utils import timezone

def get_platform_revenue_summary(start_date, end_date):
    payments = ClientPayment.objects.filter(
        paid_at__range=(start_date, end_date),
        status='paid'
    ).aggregate(
        total_payments=Sum('amount'),
        total_fees=Sum('platform_fee_amount'),
        total_payouts=Sum('consultant_payout_amount')
    )
    
    subscriptions = ConsultantSubscription.objects.filter(
        current_period_start__range=(start_date, end_date),
        status__in=['active', 'cancelled']
    ).aggregate(
        total_sub_rev=Sum('price_usd'), # Assuming USD for aggregate
        active_count=Count('id', filter=Q(status='active'))
    )
    
    payout_summary = Payout.objects.filter(
        created_at__range=(start_date, end_date)
    ).values('status').annotate(
        total=Sum('total_due')
    )
    
    # Format payout summary into a dict
    payouts = {p['status']: p['total'] for p in payout_summary}
    
    return {
        'engagement_revenue': {
            'total_client_payments': payments['total_payments'] or 0,
            'total_platform_fees': payments['total_fees'] or 0,
            'total_consultant_payouts': payments['total_payouts'] or 0
        },
        'subscription_revenue': {
            'total_subscription_revenue': subscriptions['total_sub_rev'] or 0,
            'active_subscriptions': subscriptions['active_count']
        },
        'payouts': {
            'pending': payouts.get('pending', 0),
            'approved': payouts.get('approved', 0),
            'paid': payouts.get('paid', 0)
        }
    }

from django.template.loader import render_to_string

def send_client_payment_confirmation(engagement, payment):
    context = {
        'client_name': engagement.client.contact_name,
        'engagement_title': engagement.title,
        'amount_paid': f"${payment.amount}",
        'consultant_name': engagement.consultant.full_name,
        'dashboard_url': settings.APP_URL + '/client/dashboard'
    }
    html_message = render_to_string('emails/client_payment_confirmation.html', context)
    
    send_mail(
        subject='Payment Confirmed - Your CONSSOR Engagement is Active',
        message='Payment confirmed. View your dashboard for details.',
        from_email=settings.DEFAULT_FROM_EMAIL,
        recipient_list=[engagement.client.email],
        html_message=html_message
    )

def send_consultant_engagement_notification(engagement, payment):
    context = {
        'consultant_name': engagement.consultant.full_name,
        'client_name': engagement.client.organization_name,
        'engagement_scope': ', '.join(engagement.scope),
        'your_payout': f"${payment.consultant_payout_amount}",
        'client_profile_url': f"{settings.APP_URL}/consultant/clients/{engagement.client_id}"
    }
    html_message = render_to_string('emails/consultant_engagement_notification.html', context)
    
    send_mail(
        subject='New Client Engagement Assigned',
        message='A new client has been assigned to you.',
        from_email=settings.DEFAULT_FROM_EMAIL,
        recipient_list=[engagement.consultant.email],
        html_message=html_message
    )

### Key Business Logic Rules

1. **Payment is mandatory before portal access** — No exceptions; hard-coded gate in middleware
2. **12% platform fee is non-negotiable** — Admin can override per-engagement if needed, but default is locked
3. **AI pricing confidence < 0.6** → Flag for manual admin review before sending to client
4. **Subscription grace period = 3 days** after payment failure before access lock
5. **KYC must be 100% verified** before consultant can receive payouts
6. **Referral bonus eligibility** → Only Professional and Growth plan consultants
7. **Free trial = 7 days** → Max 1 active client, max 2 lead submissions during trial
